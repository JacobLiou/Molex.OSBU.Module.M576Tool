# M576Calibrator 校准工具使用说明

## 1. 工具用途

**M576Calibrator** 用于 M576 / 1310 波长相关校准流程中的上位机辅助，主要能力包括：

- **429F 主板**：通过串口发送 **ASCII** 文本指令（如 `RECAL 1 …`），按路径表逐步执行并记录回传内容。
- **MCS（Z4671）**：使用与既有 Z4671 工具相同的 **BIN 格式** 与 **固件烧录链路**（`StartFWUpdate` → 分片传输 → `FWUpdateEnd`），将定标 BIN 写入模块。
- **1310 低温槽**：支持在生成 BIN 时，将内存中的 LUT 数据 **仅合并到低温温度槽（`IDX_TEMP_LOW`）**，其余温度点可从备份 BIN 保留。

具体命令格式、点数、超时等请以 **固件版本说明** 与 **《需求Spec PRD》** 为准；本工具对 RECAL 响应提供日志与简易数值解析，便于联调迭代。

---

## 2. 运行环境与启动

- **系统**：Windows（与工程一致，建议 Win32 程序在 64 位系统上正常运行）。
- **工程**：Visual Studio 2015（v140）、**多字节字符集（MBCS）**、MFC 动态库；编译生成可执行文件后，在本机直接运行。
- **硬件**：PC 至少具备 **两个可用串口**（或 USB 转串口），分别连接 **429F** 与 **MCS**，波特率程序内默认为 **115200，8N1**（与代码一致；若现场不同需改源码或重新编译）。

## 3. 主界面区域说明

| 区域 | 说明 |
|------|------|
| **429F COM / MCS COM** | 下拉选择串口（COM1～COM32）。两路需指向不同端口。 |
| **Open ports** | 按当前选择打开两路串口；失败时查看下方日志。 |
| **path CSV** | 校准路径表文件（见下文 CSV 格式），可用「…」浏览选择。 |
| **Backup BIN** | （可选）已有定标备份文件，用于 **合并** 时保留非低温槽数据。 |
| **Output BIN** | 生成的定标 BIN **输出路径**；「写 BIN」「烧录」均依赖此路径。 |
| **Bundle SN** | 写入 Bundle 头用的序列号；为空时程序内有默认占位，正式使用请填写真实 SN。 |
| **进度条** | 「跑路径」时按步数更新；「烧 MCS」时按分包进度更新。 |
| **日志窗口** | 显示操作结果、报错及 RECAL 回传摘要。 |

## 4. 推荐操作流程

### 4.1 连接串口

1. 接好 429F、MCS 线缆，在设备管理器中确认 COM 号。  
2. 在界面中分别选择 **429F** 与 **MCS** 对应端口。  
3. 点击 **Open ports**。成功则日志提示端口已打开。

### 4.2 准备路径 CSV

- 使用 Excel 导出的 **`path_0330` 等价 CSV**，或按下一节格式自行编辑。  
- 将完整路径填入 **path CSV**，或用「…」选择文件。

### 4.3 跑路径（RECAL）

1. 确保串口已打开（若未打开，点击 **Run path** 时会尝试自动打开）。  
2. 点击 **Run path (RECAL)**。  
3. 程序按 CSV 每一行发送 `RECAL 1`（参数由该行解析），并等待一行 ASCII 响应（单次等待约 5 s，以代码为准）。  
4. 日志中可查看每步回传；若回传为可解析的功率数组且点数为 **完全平方数**，会尝试 **十字寻峰** 并打印峰位置（辅助分析，具体含义以固件为准）。  
5. 需要中断时可点 **Stop**（对当前循环置停止标志）。

> **说明**：当前界面侧 **将校准结果写满 `m_lut` 的完整业务链** 需与工艺/固件对接后继续完善；「写 BIN」依赖内存中的 LUT 与备份合并逻辑。

### 4.4 生成 BIN（写 BIN）

1. 设置 **Output BIN** 保存路径。  
2. 若需 **保留旧 BIN 中除低温槽外的数据**，填写 **Backup BIN**。  
3. 点击 **Write BIN**。  
4. 逻辑概要：若存在有效备份则先读入 LUT，再把内存中 **1310 低温槽** 相关数据合并进去，再按 Z4671 规则写 Bundle 与 CRC；无备份则仅使用当前内存 LUT（全零或未填充时文件无实际定标意义）。

### 4.5 烧录到 MCS

1. 确认 **Output BIN** 已指向要烧录的文件且文件存在。  
2. 确保 **MCS** 串口已打开（未打开时点击烧录会尝试 **Open ports**）。  
3. 点击 **Flash MCS**。  
4. 进度条反映分包上传进度；结束后日志提示成功或失败信息。烧录结束后程序内 **等待约 5 s**（与既有工具类似），请按现场规范确认模块复位与状态。

## 5. 路径 CSV 格式

每行 **9 个整数**，逗号分隔，含义为：

```text
target_index, p1b, p1c, p2b, p2c, p3b, p3c, p4b, p4c
```

- **target_index**：目标开关序号（随 `RECAL 1` 下发）。  
- **p1b,p1c … p4b,p4c**：四段光路的 **块号（1/2）** 与 **通道号**，顺序与 PRD 中「1#1×64 → 1#MCS → 2#MCS → 2#1×64」一致。  

可选：第一行若包含英文 **target**（大小写不敏感）作为表头，会被自动跳过。  

程序内会对通道范围做 **粗校验**（例如 1×64 段 1～64、MCS 段 1～18 等），与《需求Spec PRD》路由规则一致时可减少配置错误。

示例（见工程内 `sample_path.csv`）：

```csv
target_index,p1b,p1c,p2b,p2c,p3b,p3c,p4b,p4c
1,1,1,1,1,2,1,2,1
```

## 6. 注意事项

1. **两路串口不可相同**，否则无法同时连接 429F 与 MCS。  
2. **烧录有风险**，务必先备份现场 BIN，并在工艺允许时再执行「Flash MCS」。失败时按现场流程恢复备份或重新烧录。  
3. **RECAL 语法与响应** 以固件为准；若超时或乱码，检查接线、波特率、线序及固件版本。  
4. 本工具 **V1** 聚焦 MCS（Z4671）BIN 与烧录；其它独立镜像或 1×64 专用烧录若未在工程中实现，请勿混用假设。
