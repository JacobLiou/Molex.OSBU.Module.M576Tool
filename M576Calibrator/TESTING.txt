M576 Calibrator — build and verification notes
===============================================

Build (Visual Studio 2019+, v142, Win32, MFC dynamic; Win7 SP1 target)
-----------------------------------------------------------------------
1. Open M576Calibrator\M576Calibrator.sln (ToolsVersion 16; v142 in both projects).
2. Install **MSVC v142** if missing: VS Installer → Individual components →
   "MSVC v142 - VS 2019 C++ x86/x64 build tools" (or use VS2019).
3. Configuration: Debug or Release | Win32
4. Build Z4671Core, then M576CalibratorApp

Optional: MSBuild without v142 installed — pass `/p:PlatformToolset=v143` to verify compile
on a machine that only has VS2022 default toolset (release builds for Win7 should still use v142).

Character set: **Multi-Byte** (not Unicode). The ported Z4671 sources use ANSI string
literals with CString::Format; a Unicode project would require wide literals throughout.

Topology (single COM)
---------------------
- Host opens **one** serial port to **429F** only. RECAL (ASCII) and MCS LUT programming
  (Z4671 binary frames) share the same `Z4671Command` instance; firmware forwards binary
  traffic to MCS.
- On bring-up: run a short RECAL path, then flash a small known-good BIN on the **same**
  COM; confirm 429F firmware version supports transparent forward.

CRC / BIN parity (manual)
-------------------------
- CLutBinWriter::Write follows the same fwrite order and CRC steps as legacy CreateBinFileZ4671
  (stLutBundleHeader1, stLutBundleHeader2, stImageHeader x2, stLutSettingZ4671).
- Regression: fill an identical stLutSettingZ4671 in the old tool and in this tool, write BIN,
  compare file hashes.

Integration checklist
---------------------
429F (ASCII + forward)
- Open the single 429F COM, run a short path CSV; confirm RECAL lines and responses in the log.
- Tune ReadAsciiResponse timeout and line framing when firmware samples are available.

MCS LUT (Z4671 binary via 429F)
- Use "烧录定标" with a known-good LUT BIN; confirm StartFWUpdate → transport → FWUpdateEnd
  on the same link (no separate MCS COM port on PC).

path_0330 CSV format (nine integer fields per row)
--------------------------------------------------
Default file: .\\output\\standard.csv (see tools/generate_sample_path_1286.py).
target_index, p1b, p1c, p2b, p2c, p3b, p3c, p4b, p4c
Optional first row with header containing “target” is skipped.

1310 merge
----------
- Backup BIN optional: ReadLutFromFile → MergeLut1310LowTempSlot → CLutBinWriter::Write
- Only IDX_TEMP_LOW (0) wCalibPtrDAC and per-switch low temperature point are overwritten from session LUT.
